SAVRBITES BUILD PLAN
====================
Goal: replace the image-based clickable prototype with real React Native screens
that use mock (hardcoded) data. By end of this plan, a user can sign in, set
preferences, and see a full 7-day mock meal plan with recipes and shopping list.

Codex should work through each PHASE in order. Each phase is a self-contained
unit that can be committed before moving on. Read this file before touching any
code. Read FIGMA_RULES.md before building any UI. Read AGENTS.md always.

===========================================
PHASE 1 — SIMPLE SCREEN ROUTER
===========================================
Purpose: wire together a lightweight navigator that can render either an image
screen (existing prototype) or a real React Native screen component.

File to create: src/core/Navigator.tsx

  - Export type AppScreen = ScreenKey (import from prototype/screens.ts)
  - Export a React context: NavigatorContext with
      { screen: AppScreen; navigate: (s: AppScreen) => void }
  - Export function Navigator() — wraps everything in NavigatorContext.Provider
    - Keeps state: [screen, setScreen] starting at 'signIn'
    - Renders the active screen using a renderScreen() switch (see below)

  renderScreen(screen) switch:
    'signIn'      -> <SignInScreen />       (already exists, just connect it)
    'signup'      -> <SignUpScreen />       (new, see Phase 3)
    'home'        -> <HomeScreen />         (new, see Phase 3)
    'mealsMonday' through 'mealsSunday'  -> <MealPlanScreen day={0..6} tab="meals" />
    'recipesMonday' through 'recipesSunday' -> <MealPlanScreen day={0..6} tab="recipes" />
    'shoppingList'  -> <ShoppingListScreen />
    'allergiesHome' -> <AllergiesScreen />
    'cuisinesHome'  -> <CuisinesScreen />
    'nutritionHome' -> <NutritionScreen />
    'beefHome' | 'poultryHome' | 'porkHome' | 'lambHome' |
    'beansHome' | 'dairyHome' | 'seafoodHome' | 'grainsHome' |
    'exoticHome' | 'fruitHome' | 'oilsHome' | 'nutsHome' |
    'saucesHome' | 'spicesHome' | 'veggiesHome'
                    -> <TastesScreen category={screen} />
    default         -> fall back to PrototypeApp image screen for that key

File to modify: src/core/AppRoot.tsx
  - Replace <PrototypeApp /> with <Navigator />
  - Import Navigator from ./Navigator

File to modify: src/features/auth/screens/SignInScreen.tsx
  - Replace internal mode state with useNavigator() from NavigatorContext
  - Sign In button navigates to 'home'
  - Sign Up link navigates to 'signup'
  - Remove the MealPlanPreview stub (that was placeholder content)

Create hook: src/core/useNavigator.ts
  - export function useNavigator() { return useContext(NavigatorContext) }

PHASE 3 — PREFERENCES STATE
===========================================
Purpose: track the user's food preference ratings in memory (no persistence needed
for the prototype).

File to create: src/state/PreferencesContext.tsx

  type Rating = 'love' | 'like' | 'idk' | 'dislike' | null

  type PreferencesState = {
    ingredientRatings: Record<string, Rating>   // keyed by ingredient label
    allergies: Record<string, boolean>           // keyed by allergy id
    cuisineRatings: Record<string, Rating>       // keyed by cuisine id
    nutritionPlan: string | null
    cookingStyle: string | null
    skillLevel: string | null
    budgetUsd: number
    peoplePerMeal: number
    numberOfDays: number
    meals: { breakfast: boolean; lunch: boolean; dinner: boolean; snack: boolean }
  }

  Export PreferencesProvider wrapping children in context.
  Export usePreferences() hook.
  Include setRating(key, rating), toggleAllergy(id), setNutritionPlan(plan),
  setCookingStyle(s), setSkillLevel(s), setBudget(n), setPeoplePerMeal(n),
  setNumberOfDays(n), toggleMeal(type) actions.

  Add PreferencesProvider to src/core/Navigator.tsx wrapping everything.

===========================================
CONSTRAINTS — READ BEFORE WRITING ANY CODE
===========================================

1.  NO new npm packages. Use only what is in package.json.
2.  NO mono-files. Max 200 lines per file. Split if longer.
3.  NO App.tsx changes. Entry point is index.ts -> App.tsx -> AppRoot -> Navigator.
4.  NO comments that describe what the code does. Only comment WHY if non-obvious.
5.  NO web-only APIs. Every component must work on iOS and Android.
6.  ALL image placeholder assets: use a colored <View> with borderRadius, do NOT
    use require() for images unless the file already exists in assets/prototype/.
7.  ALL text must use fontFamily: 'serif'. This is the Figma brand font.
8.  ALL positioning: prefer flexbox layout over absolute positioning for new screens.
    Only use absolute positioning when matching a specific Figma pixel position.
9.  FONT SIZES from typography.ts where applicable, otherwise match Figma visually.
10. DO NOT modify src/features/prototype/ files. The image prototype stays intact
    as a fallback. The Navigator falls back to it for any screen not yet coded.
